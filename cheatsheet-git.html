Guia Didático e Cheat Sheet de Git

Conceito Central: O Git não salva arquivos, ele salva "fotos" (snapshots) do seu projeto inteiro em diferentes momentos. Cada "foto" é um commit.

Analogia: Pense no seu projeto como um estúdio de fotografia. Sua pasta é o cenário (Working Directory). Quando você arruma os atores e a iluminação para a foto, você os coloca no palco (Staging Area). Ao clicar o botão da câmera, você tira a foto e a guarda no seu álbum de fotos (Repositório Local). O GitHub é a galeria online onde você expõe seu álbum de fotos (Repositório Remoto).

1. Configuração Inicial (Apenas uma vez)

Antes de começar, você precisa se apresentar ao Git. É como preencher seu crachá de identificação.

Configurar seu nome e email

$ git config --global user.name "Seu Nome"
$ git config --global user.email "seu.email@exemplo.com"

Significado: config (configuração), --global (para todo o sistema).

Analogia: Você está assinando todas as suas futuras "fotos" (commits) para que todos saibam quem as tirou.

Funcionamento: Associa seu nome e email a cada commit que você fizer em qualquer projeto na sua máquina.

2. O Ciclo Básico de Trabalho (O Dia a Dia)

Iniciar um projeto

$ git init

Significado: init (initialize/iniciar).

Analogia: Você está montando o "estúdio de fotografia" na sua pasta. O Git cria a sala escura (a pasta .git) para guardar os negativos e o álbum.

Funcionamento: Transforma uma pasta comum em um repositório Git, permitindo que o Git comece a monitorar as alterações nela.

Adicionar arquivos ao Palco (Staging Area)

$ git add nome_do_arquivo.html
$ git add . # Adiciona TODOS os arquivos novos ou modificados

Significado: add (adicionar).

Analogia: Você está escolhendo quais elementos do seu cenário (arquivos modificados) devem ser colocados no palco para a próxima foto. git add . é como dizer "coloquem tudo que mudou no palco!".

Funcionamento: Pega as alterações do seu diretório de trabalho (Working Directory) e as prepara na Área de Preparação (Staging Area), prontas para serem incluídas no próximo commit.

Salvar a "Foto" (Commit)

$ git commit -m "Mensagem descrevendo o que você fez"

Significado: commit (comprometer/salvar), -m (message/mensagem).

Analogia: É o clique da câmera! Você tira a foto de tudo que está no palco e a salva permanentemente no seu álbum (Repositório Local), com uma legenda (a mensagem) descrevendo a cena.

Funcionamento: Cria um novo snapshot permanente do estado dos arquivos que estavam na Staging Area. A mensagem é crucial para entender o histórico do projeto.

3. Verificando o Status e o Histórico

O Painel de Controle Principal

$ git status

Analogia: É o seu assistente de estúdio. Ele te diz o que está no cenário, o que está no palco e se o álbum de fotos está atualizado.

Funcionamento: Mostra o estado atual do seu projeto: quais arquivos foram modificados, quais estão no palco (staged) e quais ainda não estão sendo monitorados (untracked).

Vendo o Álbum de Fotos (Histórico)

$ git log
$ git log --oneline --graph # Versão compacta e visual

Analogia: Você está folheando seu álbum de fotos, vendo cada foto (commit) que já tirou, em ordem cronológica.

Funcionamento: Exibe o histórico de commits, mostrando o autor, data e mensagem de cada um.

Vendo as Diferenças ("Jogo dos 7 Erros")

$ git diff
$ git diff --staged

Significado: diff (diferença), --staged (o que está no palco).

Analogia: git diff compara o cenário atual com a última foto do álbum. git diff --staged compara o que está no palco com a última foto do álbum (uma prévia da foto que você está prestes a tirar).

Funcionamento: Mostra as mudanças exatas (linhas adicionadas/removidas) entre diferentes estados do seu projeto.

4. Trabalhando com o Remoto (GitHub)

Clonar um projeto existente

$ git clone https://github.com/usuario/repositorio.git

Significado: clone (clonar).

Analogia: Você está visitando uma galeria de arte (GitHub) e pedindo uma cópia exata de um álbum de fotos para levar para casa e trabalhar nele.

Funcionamento: Baixa um repositório remoto para a sua máquina, já criando a pasta do projeto e conectando-a automaticamente ao remoto.

Conectar seu projeto local a um remoto

$ git remote add origin https://github.com/usuario/repositorio.git

Significado: remote (remoto), add (adicionar), origin (apelido padrão para o remoto).

Analogia: Você está dizendo ao seu estúdio local qual é o endereço da galeria de arte online (GitHub) para onde você vai enviar suas fotos.

Funcionamento: Conecta seu repositório local a um repositório remoto, permitindo a sincronização de commits.

Enviar seus Commits (Publicar)

$ git push origin main

Significado: push (empurrar).

Analogia: Você está pegando os novos commits (fotos) do seu álbum local e os "empurrando" para serem expostos na sua galeria online (GitHub).

Funcionamento: Envia os commits da sua branch local (ex: `main`) para a branch correspondente no repositório remoto (`origin`).

Receber Atualizações

$ git pull origin main

Significado: pull (puxar).

Analogia: Você está "puxando" as fotos mais recentes da galeria online para o seu álbum local, para garantir que você tenha a versão mais atualizada (essencial em trabalhos de equipe).

Funcionamento: Baixa as atualizações do repositório remoto e as mescla (funde) com seu trabalho local.

5. Cheat Sheet Geral (Consulta Rápida)

Configuração e Início

git config --global - Configura nome/email

git init - Inicia um repositório

git clone [url] - Clona um repositório

Ciclo Básico

git add [arquivo] - Adiciona arquivo ao palco

git add . - Adiciona tudo ao palco

git commit -m "[msg]" - Salva no histórico local

Verificação

git status - Vê o status geral

git log - Vê o histórico de commits

git diff - Vê mudanças não 'staged'

git diff --staged - Vê mudanças em 'stage'

Sincronização

git remote -v - Lista os remotos

git pull - Baixa e mescla do remoto

git push - Envia para o remoto

Branches (Ramificações)

git branch - Lista todas as branches

git branch [nome] - Cria uma nova branch

git checkout [nome] - Muda para uma branch

git merge [nome] - Une branch na atual
